# Clothes（工服/衣物扫码分拣系统）

[English](README.md) · **简体中文** · [日本語](README.ja.md)

一个移动端优先的 Web App，用于工服仓库、制服分发、衣物打包分拣等场景。

> “简而不简，扫码即达。”

---

## ✨ 核心特性

- 📱 **移动端优先** — 针对仓库现场的手机操作优化，按钮大、反馈清晰。
- 🔍 **扫码或手动输入** — 支持二维码扫描或手动输入衣物编号，例如 `ITEM-000001`。
- 🏷️ **目标位置指引** — 根据尺码自动推荐分拣位置，例如 M → `A-01`、L → `A-02`、XL → `A-03`。
- 💾 **本地数据持久化** — 使用 `localStorage` 存储数据，无需数据库、后端或网络服务。
- 🧪 **测试覆盖关键路径** — 使用 Vitest 覆盖数据生成、路由规则、异常状态和边界行为。

## 📥 批量导入格式

当前版本通过 **初始化示例数据** 按钮快速生成演示数据。

示例批次信息如下：

```text
批次: BATCH-20260704-BLUE
数量: 30 件
类型: 蓝色制服上衣
状态: 待分拣
初始位置: 待分拣区
```

## 🚀 下载与使用

### 1. 获取程序

```bash
git clone <repository-url>
cd Clothes
```

### 2. 安装依赖

```bash
npm install
```

如果 npm 官方源网络不稳定，可以临时使用镜像源：

```bash
npm install --registry=https://registry.npmmirror.com
```

### 3. 启动开发服务器

```bash
npm run dev
```

默认打开：

```text
http://localhost:3000
```

如果 `3000` 端口被占用，可以指定其他端口：

```bash
npm run dev -- -p 3001
```

## 📋 使用流程

1. 打开首页。
2. 点击 **初始化示例数据**。
3. 点击 **开始分拣** 进入 `/scan`。
4. 输入衣物编号，例如 `ITEM-000001`。
5. 按 Enter 或点击查询。
6. 页面会显示衣物信息和醒目的目标位置。
7. 点击 **确认放入目标位置**，衣物状态会变成 `已分拣`。
8. 再次扫描已经分拣的衣物时，会提示重复操作。

## 🗂️ 页面导航

| 路由 | 说明 |
| :--- | :--- |
| `/` | 首页统计、当前批次、初始化数据、清空数据、页面入口 |
| `/scan` | 分拣页：输入编号、确认分拣、标记异常、修改目标位置 |
| `/items` | 衣物列表：搜索和状态筛选 |
| `/items/[itemCode]` | 衣物详情：编辑姓名、部门、状态、位置、备注 |
| `/locations` | 位置管理：新增和删除位置；正在被衣物使用的位置不能删除 |
| `/logs` | 扫码和操作记录，按时间倒序显示 |

## 🧪 测试覆盖

测试重点不是只测 happy path，而是覆盖容易作弊或漏掉的行为：

- 示例数据必须生成 30 件唯一 item
- M / L / XL 必须分别路由到 `A-01` / `A-02` / `A-03`
- `localStorage` 未初始化时返回安全空数据
- 已分拣和异常状态会正确影响统计
- 异常不计入完成率
- 更新不存在的 item 不会创建幽灵数据
- 重复扫码和重复确认分拣必须记录日志
- 正在被衣物引用的位置不能删除
- 清空数据只清理本应用自己的 storage key
- 页面和组件不能绕过 `lib/storage.ts` 直接调用 `localStorage`
