# Clothes（ユニフォーム・衣類 QR 仕分けシステム）

[English](README.md) · [简体中文](README.zh-CN.md) · **日本語**

モバイルファーストの Web アプリ。ユニフォーム倉庫、制服配布、衣類の仕分け・梱包現場などで使用します。

> 「シンプルでありながら十分。スキャンすれば、目的の場所へ。」

---

## ✨ 主な特徴

- 📱 **モバイルファースト** — 倉庫現場でのスマホ操作に最適化。大きなボタンと明確なフィードバック。
- 🔍 **スキャンまたは手動入力** — QR コードスキャンまたは `ITEM-000001` のような衣類番号の手動入力に対応。
- 🏷️ **目標場所の案内** — サイズに応じて仕分け場所を自動提案。M → `A-01`、L → `A-02`、XL → `A-03`。
- 💾 **ローカルデータの永続化** — `localStorage` でデータを保存。データベースやバックエンド、ネットワークサービスは不要。
- 🧪 **重要パスのテストカバレッジ** — Vitest でデータ生成、ルーティングルール、例外状態、境界値をカバー。

---

## 📥 インポート形式

現在のバージョンは **サンプルデータの初期化** ボタンでデモデータを迅速に生成します。

サンプルバッチ情報：

```text
バッチ: BATCH-20260704-BLUE
数量: 30 着
種類: 青色制服上衣
状態: 未仕分け
初期位置: 未仕分けエリア
```

## 🚀 ダウンロードと使い方

### 1. アプリの取得

```bash
git clone <repository-url>
cd Clothes
```

### 2. 依存関係のインストール

```bash
npm install
```

公式 npm レジストリが遅い場合はミラーを使用：

```bash
npm install --registry=https://registry.npmmirror.com
```

### 3. 開発サーバーの起動

```bash
npm run dev
```

デフォルト URL：

```text
http://localhost:3000
```

ポート `3000` が使用中の場合は別のポートを指定：

```bash
npm run dev -- -p 3001
```

## 📋 利用フロー

1. ホームページを開く。
2. **サンプルデータの初期化** をクリック。
3. **仕分け開始** をクリックして `/scan` に移動。
4. `ITEM-000001` などの衣類番号を入力。
5. Enter キーまたは検索ボタンを押す。
6. 衣類情報と目標場所が表示される。
7. **目標場所に入れる** をクリック。
8. 再スキャンすると重複操作の警告が表示される。

## 🗂️ ページナビゲーション

| ルート | 説明 |
| :--- | :--- |
| `/` | ホーム統計、現在のバッチ、データ初期化・削除、ページ入口 |
| `/scan` | 仕分けページ：番号入力、仕分け確認、異常マーク、目標場所変更 |
| `/items` | 衣類一覧：検索と状態フィルタ |
| `/items/[itemCode]` | 衣類詳細：名前、部門、状態、位置、備考の編集 |
| `/locations` | 位置管理：位置の追加・削除；使用中の位置は削除不可 |
| `/logs` | スキャンと操作ログ、新しい順 |

## 🧪 テストカバレッジ

テストは happy path だけでなく、見逃しやすい挙動もカバーします：

- サンプルデータは 30 件のユニークな item を生成する必要がある。
- M / L / XL はそれぞれ `A-01` / `A-02` / `A-03` にルーティングされる必要がある。
- 未初期化の `localStorage` は安全な空データを返す必要がある。
- 仕分け済みと異常状態が統計に正しく反映される必要がある。
- 異常は完成率に含めてはいけない。
- 存在しない item の更新でゴーストデータを作ってはいけない。
- 重複スキャンと重複仕分け確認はログに記録される必要がある。
- 衣類に使用中の位置は削除できない。
- データクリアはこのアプリ専用の storage key のみを削除する。
- ページやコンポーネントは `localStorage` を直接呼ばず、`lib/storage.ts` を経由する。
